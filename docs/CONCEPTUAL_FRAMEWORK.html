System Overview

AgrIDirect is a modern agricultural e-commerce platform designed to connect farmers and agricultural producers directly with consumers and buyers. Built with Flutter for cross-platform mobile (iOS & Android) and web deployment, the system leverages Supabase as a Backend-as-a-Service with PostgreSQL for data persistence, Firebase for authentication, and PayMongo/PayMaya for payment processing. It provides a complete ecosystem for product listings, order management, pre-ordering, payment processing, community interaction, real-time messaging, weather monitoring, and data analytics. The platform features offline-first capability through SQLite caching, geolocation services for location-based features, and camera integration with ML Kit for face detection.

Framework Components

Infrastructure Layer

☁️ Internet

  • Primary communication channel via HTTPS/REST
  • Connects clients to Supabase, Firebase, and payment gateway services
  • Enables real-time data synchronization via Supabase Realtime

🖥️ Supabase Server (BaaS)

Role: Backend-as-a-Service Provider

Functions:

  • RESTful API communication hub
  • Real-time database subscriptions
  • Database connection management
  • Data validation and business logic execution
  • Storage for product images and user uploads

🗄️ PostgreSQL Database (32 Tables + 7 Views)

Role: Primary Data Persistence Layer

Tables include: users, products, orders, order_items, reviews, forum_posts, forum_comments, farmer_profiles, farmer_registrations, admin_records, payments, pre_orders, messages, notifications, wishlist, and more

Features:

  • 3NF normalized design
  • Row Level Security (RLS) on all tables
  • Role-based access control (consumer, seller, admin)
  • Transactional integrity and data consistency
  • 7 computed views for analytics and reporting

🔐 Firebase Auth & Google Sign-In

Role: Authentication & Authorization

Functions:

  • User authentication and session management
  • OAuth 2.0 integration with Google Sign-In
  • OTP verification for account security
  • Secure credential storage
  • Multi-device sign-in support

💳 Payment Gateway (PayMongo & PayMaya)

Role: Payment Processing

Functions:

  • Online payment processing via PayMongo API
  • PayMaya integration as alternative payment method
  • WebView-based payment checkout flow
  • Payment status tracking and verification
  • Support for regular orders and pre-order payments

Data Flow & Process Workflows

Product Publishing Flow

Farmer captures product images using device camera
Farmer creates product listing with details (name, price, category, quantity, unit type)
System validates data and stores in PostgreSQL via Supabase
Product is indexed and made searchable in the Marketplace
Consumers can discover, search, filter, and add to wishlist

Order & Payment Flow

Consumer browses marketplace and adds products to cart
Consumer proceeds to checkout with delivery address
Payment is processed via PayMongo/PayMaya through WebView checkout
System creates order record, farmer receives notification
Consumer tracks order status (pending → confirmed → delivered)

Pre-order Flow

Consumer browses Pre-order Hub for upcoming products
Consumer creates pre-order with desired quantity
Pre-order payment is processed via PayMongo
Farmer is notified and confirms pre-order fulfillment
Product is delivered when available, status updated in real-time

Community Engagement Flow

User creates forum post or comment in Community Hub
System stores content in PostgreSQL and indexes it
Other users discover, engage, comment, and like content
Admin monitors and moderates content via Moderation tab
Community knowledge and insights are aggregated

Offline Capability & Caching

AgrIDirect leverages SQLite local caching with a dedicated Offline Sync Service for offline-first operation:

  • Product listings and user data are cached locally on the device
  • Users can browse products and compose orders without internet
  • Changes are queued and automatically synced when connectivity is restored
  • Data consistency is maintained across all devices through Supabase sync
  • Connectivity Plus plugin detects network status changes in real-time
